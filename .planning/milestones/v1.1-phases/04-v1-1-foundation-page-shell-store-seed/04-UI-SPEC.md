## Spacing Scale

Declared values (must be multiples of 4). Source: established Tailwind classes observed across `RecipeWorkbenchPage.tsx`, `HomePage.tsx`, `MissingFlagBanner.tsx`.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (`p-1`, `gap-1`) | Icon-text gaps, fine inline spacing |
| sm | 8px (`p-2`, `gap-2`, `space-x-2`) | Compact element padding, icon gutters |
| md | 16px (`p-4`, `gap-4`, `px-4`) | Default card padding, section inner padding |
| lg | 24px (`p-6`, `gap-6`) | Card/panel padding, between-section gaps |
| xl | 32px (`py-8`, `mb-8`) | Page header bottom margin, major section breaks |
| 2xl | 48px (`py-12`) | Full-page vertical breathing room on spacious screens |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Touch targets (mobile hamburger button, nav links in mobile drawer): minimum 44px height via `py-2` + `text-base` line-height вҖ” matches existing mobile nav in `AppRouter.tsx`.
- Meal-plan column fixed width: `lg:w-80` (320px) to `lg:w-96` (384px) вҖ” executor chooses one value; 320px is the lower bound per CONTEXT.md.

## Typography

Source: observed classes in `RecipeWorkbenchPage.tsx`, `HomePage.tsx`, `AppRouter.tsx`.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (`text-sm`) | 400 (`font-normal` / default) | 1.5 (Tailwind default for `text-sm`) |
| Label / nav link | 14px (`text-sm`) вҖ” desktop nav; 16px (`text-base`) вҖ” mobile nav | 400 (`font-normal` / default) | 1.5 |
| Heading (page title) | 30px (`text-3xl`) | 700 (`font-bold`) | 1.2 (`leading-tight` or browser default) |
| Subheading / tagline | 16px (`text-base`) | 400 (`font-normal`) | 1.5 |

Sizes are exactly 4 values. Weights are exactly 2 (400 regular + 700 bold). Nav link distinction from body text is achieved via color contrast (`text-gray-600 dark:text-gray-300`) rather than weight вҖ” this matches the plain-weight nav pattern used across existing pages. All sizes match what is already in use project-wide; no new type sizes are introduced in Phase 4.

## Color

Source: `chat/tailwind.config.js` (primary palette), `MissingFlagBanner.tsx` (alert yellow), `RecipeWorkbenchPage.tsx`, `HomePage.tsx` (surface colors and text colors). `darkMode: 'class'` вҖ” all color tokens below have a `dark:` counterpart.

| Role | Light value | Dark value | Usage |
|------|-------------|------------|-------|
| Dominant (60%) | `white` / `gray-50` (`bg-white`) | `gray-800` (`dark:bg-gray-800`) | Page background, main surface |
| Secondary (30%) | `white` with `border-gray-200` (`bg-white border`) | `gray-800` with `border-gray-700` (`dark:bg-gray-800 dark:border-gray-700`) | Cards, column panels (`MealPlanColumn`, chat placeholder panel) |
| Accent (10%) | `primary-500` (#3b82f6) | `primary-400` (#60a5fa) | Gradient icon block in page header only; primary CTA `Add to Plan` button (Phase 6 вҖ” not rendered in Phase 4) |
| Alert / banner | `yellow-50` + `yellow-200` border | `yellow-900/20` + `yellow-800` border | `MissingFlagBanner` вҖ” reused as-is, no changes |
| Destructive | `red-600` / `red-500` | `red-400` | Remove-from-plan button (Phase 4 renders no plan entries, so no destructive UI appears in this phase; token reserved for Phase 6) |

Accent reserved for:
1. Page header icon block background (gradient `from-primary-500 to-purple-600`) вҖ” one element per page, matching the `RecipeWorkbenchPage` and `HomePage` pattern
2. Focus rings (`focus:ring-primary-500`) on all interactive elements

The accent color does NOT apply to: body text, card borders, labels, empty-state copy, column backgrounds, or the `MissingFlagBanner`.

## Interaction Contract

### Nav Link

- **Desktop:** `<Link to="/generative-ui">` inserted immediately after the `/webmcp` `<Link>` in `AppRouter.tsx` desktop nav block (lines ~70вҖ“72). Class: `text-gray-600 dark:text-gray-300 hover:text-gray-900 dark:hover:text-white px-3 py-2 rounded-md text-sm font-normal transition-colors duration-200`. Includes `onClick={() => trackUserInteraction('navigation_click', 'generative_ui_link')}`.
- **Mobile:** Identical `<Link>` in mobile nav block (lines ~160вҖ“162). Class adds `block` prefix. Includes matching `trackUserInteraction` call.

### MissingFlagBanner

- Renders at top of `GenerativeUIPage` when `typeof navigator.modelContext === 'undefined'`.
- Does NOT unmount or hide when the user scrolls вҖ” it stays above the two-column layout.
- The rest of the page (column shell, empty states) remains fully visible and usable while banner is shown.
- No dismiss/close affordance вҖ” matches v1.0 behavior exactly.

### ChatPlaceholder

- Static, non-interactive. No buttons, no inputs.
- Renders the empty-state heading and body copy only.
- Visual treatment: `bg-white dark:bg-gray-800 rounded-xl shadow-sm border border-gray-200 dark:border-gray-700 p-6` вҖ” matches existing card style.
- Minimum height: `min-h-[400px]` on desktop to prevent the left column from collapsing before Phase 6 adds real content.

### MealPlanColumn

- Renders empty-state when `useMealPlan()` returns an empty array.
- Empty-state visual treatment: centered flex column, muted icon (inline SVG, gray-400), empty-state heading + body in `text-gray-500 dark:text-gray-400`.
- No interactive affordances in Phase 4 (no add/remove buttons вҖ” those ship in Phase 6).
- Subscribes to `MealPlanStore` via `useMealPlan()` hook вҖ” ready to re-render when Phase 6 writes entries, no refactor needed.
- Loading state: while IDB `getPlan()` is in-flight, render a single line `text-gray-500 dark:text-gray-400 text-sm` reading `Loading planвҖҰ` вҖ” matches the RecipeWorkbench loading copy pattern.
