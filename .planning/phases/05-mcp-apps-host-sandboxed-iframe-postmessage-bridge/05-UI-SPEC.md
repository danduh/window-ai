# Phase 5 — UI Design Contract

> Visual and interaction contract for Phase 5: MCP Apps Host — Sandboxed iframe + postMessage bridge.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — Tailwind CSS 3.4.3 with custom `primary` color scale (source: `chat/tailwind.config.js`) |
| Preset | not applicable |
| Component library | none — inline JSX components, no Radix/Base UI |
| Icon library | none — inline SVG (project-wide convention) |
| Font | system-ui stack (inherited from browser; no custom font loaded) |

No `components.json` found in project root. New components follow the established hand-rolled Tailwind TSX pattern exactly. No new design-system tooling is introduced in this phase.

**Two design systems coexist in this phase:**
- **Outer (React/host):** Tailwind v3 via PostCSS — standard project conventions. Dark mode via `dark:` class prefix with `ThemeProvider`.
- **Inner (iframe srcdoc):** Hand-written inline `<style>` — a small Tailwind-like utility subset extracted manually. The iframe cannot load `tailwind.css` (null origin, `default-src 'none'` CSP). Dark mode via `[data-theme="dark"]` attribute selectors only (NOT `prefers-color-scheme` — sandboxed iframes do not inherit the parent's color scheme).

## Spacing Scale

Inherited from Phase 4. No new tokens introduced.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (`p-1`, `gap-1`) | Icon-text gaps, fine inline spacing |
| sm | 8px (`p-2`, `gap-2`, `space-x-2`) | Compact element padding, icon gutters |
| md | 16px (`p-4`, `gap-4`, `px-4`) | Default card padding, section inner padding |
| lg | 24px (`p-6`, `gap-6`) | Card/panel padding, between-section gaps |
| xl | 32px (`py-8`, `mb-8`) | Page header bottom margin, major section breaks |
| 2xl | 48px (`py-12`) | Full-page vertical breathing room |
| 3xl | 64px | Not used in this phase |

Phase 5 additions:
- `ChatBubbleContainer` inner padding: `p-2` (8px) — intentionally tight so the iframe content reads as "inside" the bubble without excessive whitespace.
- `UIResourceFrame` initial height: 320px — set via inline style `style={{ height: '320px' }}` until the first `ui/notifications/size-changed` fires.
- Carousel card padding (iframe): 16px (`padding: 1rem`) per card. Gap between cards: 12px (`gap: 0.75rem`) — falls between sm and md; justified because card row scrolls horizontally and 12px is the tightest readable gap for this content density.

Exceptions: none beyond the above.

## Typography

Inherited from Phase 4. No new type sizes or weights.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (`text-sm`) | 400 (`font-normal`) | 1.5 |
| Label / nav link | 14px (`text-sm`) desktop; 16px (`text-base`) mobile | 400 (`font-normal`) | 1.5 |
| Heading (page title) | 30px (`text-3xl`) | 700 (`font-bold`) | 1.2 |
| Subheading / tagline | 16px (`text-base`) | 400 (`font-normal`) | 1.5 |

**Iframe-side typography (inline CSS subset):**

| Role | CSS | Weight | Line Height |
|------|-----|--------|-------------|
| Card title | `font-size: 0.9375rem` (15px) | `font-weight: 600` | `1.3` |
| Card meta (minutes badge, ingredient text) | `font-size: 0.8125rem` (13px) | `font-weight: 400` | `1.4` |
| "Pick" button label | `font-size: 0.875rem` (14px) | `font-weight: 600` | `1` |

The iframe uses a 2-weight system: 400 (normal) and 600 (semibold). 600 replaces the outer system's 700 bold because card titles at 15px are more readable at semibold than full bold. No italic, no other weights.

## Interaction Contract

### "Show demo carousel" button

- **Placement:** inside `ChatPlaceholder`, immediately below the body copy paragraph, separated by `mt-6`.
- **State:** single-slot — clicking once mounts one `UIResourceFrame`. Clicking again while the frame is mounted does nothing (button becomes disabled or hidden). Re-clicking after an error state resets and re-mounts.
- **Class contract:**
  ```
  mt-6 inline-flex items-center gap-2 px-4 py-2 rounded-lg
  bg-primary-600 hover:bg-primary-700 text-white text-sm font-semibold
  transition-colors duration-200 focus:outline-none focus:ring-2
  focus:ring-primary-500 focus:ring-offset-2 dark:focus:ring-offset-gray-800
  disabled:opacity-50 disabled:cursor-not-allowed
  ```
- **Font weight:** `font-semibold` (600) — matches `LiveTranslatePage.tsx` button pattern; sets the standard for Phase 6 buttons on this page.
- **Icon:** optional inline SVG chevron-right (16×16, `currentColor`) to the left of the label — executor's call. If included: `<svg class="w-4 h-4" ...>`. If omitted, the button still satisfies the spec.
- **Disabled state:** button disables (`disabled` attribute) while the iframe is mounting (loading state). Re-enables only on error (so user can retry) or if the iframe is explicitly closed.
- **Focus ring:** `focus:ring-2 focus:ring-primary-500 focus:ring-offset-2` — matches project-wide focus pattern.

### ChatBubbleContainer

- **Mounting:** appended directly below the `ChatPlaceholder` inner div when the user clicks "Show demo carousel". Uses `{showFrame && <ChatBubbleContainer>...</ChatBubbleContainer>}` conditional in `ChatPlaceholder`.
- **Animation:** `animate-fade-in` (Tailwind custom animation, 0.5s ease-in-out, defined in `tailwind.config.js`). Applied to the outer `ChatBubbleContainer` div on mount.
- **No interaction affordances** on the container itself. The container is purely a visual frame.

### UIResourceFrame — iframe lifecycle states

Three mutually exclusive visual states inside the `ChatBubbleContainer`:

**1. Loading state (from iframe mount until handshake completes, or 200ms minimum):**

```
<div
  aria-label="Loading recipe carousel…"
  class="h-[200px] rounded-xl bg-gray-200 dark:bg-gray-700 animate-pulse"
/>
```

A single full-width pulsing skeleton bar at 200px height. No spinner, no text — keeps the visual minimal while the bridge initializes.

**2. Error state (handshake timeout — 1000ms elapsed, no `ui/initialize` received):**

```
<div class="rounded-xl border border-red-300 dark:border-red-700
            bg-red-50 dark:bg-red-900/20 p-4 flex flex-col gap-3">
  <p class="text-sm font-semibold text-red-700 dark:text-red-400">
    Couldn't load app
  </p>
  <p class="text-sm text-red-600 dark:text-red-300">
    The recipe carousel didn't respond in time. This can happen if WebMCP
    isn't enabled in your browser.
  </p>
  <button class="self-start px-3 py-1.5 text-sm font-semibold rounded-md
                 bg-red-600 hover:bg-red-700 text-white transition-colors
                 duration-200 focus:outline-none focus:ring-2
                 focus:ring-red-500 focus:ring-offset-2">
    Try again
  </button>
</div>
```

"Try again" destroys the current iframe and remounts it fresh (new srcdoc assignment, new bridge instance).

**3. Success state (handshake complete — `ui/notifications/initialized` received):**

The `<iframe>` element is shown at full width. Height adjusts via `ui/notifications/size-changed` up from the 320px default. The loading skeleton is unmounted.

### "Pick" button interaction (inside iframe)

- **Default state:** `background: var(--c-btn-bg); color: var(--c-btn-text); border-radius: 0.375rem; padding: 0.375rem 0.75rem; font-size: 0.875rem; font-weight: 600; cursor: pointer; border: none; width: 100%; transition: background 0.15s;`
- **Hover:** `background: var(--c-btn-hover)`
- **Focus:** `outline: 2px solid var(--c-btn-bg); outline-offset: 2px`
- **In-flight (after click, before JSON-RPC response):** button disabled, text stays `Pick`, opacity: 0.6, `cursor: not-allowed`
- **Success (JSON-RPC response received):** button text changes to `Added!`, background stays accent, no disabled state. After 2000ms the button text reverts to `Pick` and re-enables.
- **Error (JSON-RPC error or navigator.modelContext unavailable):** button text changes to `Tools unavailable`, background becomes `#6b7280` (gray-500), disabled. Does not revert — permanent in this session (user sees the banner and knows why).

### Theme propagation

- `UIResourceFrame` consumes `useTheme()` from `ThemeContext`.
- On initial mount: immediately posts `ui/notifications/host-context-changed` with `{ theme: currentTheme }` after the iframe `load` event fires (via the pre-load outbound queue).
- On theme change: posts `ui/notifications/host-context-changed` with the new theme value.
- Iframe-side handler: sets `document.documentElement.setAttribute('data-theme', theme)` on `<html>`.
- The `[data-theme="dark"]` CSS selector block activates the dark palette.
- **No flash-of-wrong-theme:** the initial host-context post is queued before `load` fires (pre-load queue), so the iframe receives the current theme as part of the handshake flow before it paints.

### Height sync (ResizeObserver)

- Iframe-side inline script attaches `new ResizeObserver(entries => { ... })` to `document.documentElement`, debounced 50ms.
- On observation: posts `{ jsonrpc: '2.0', method: 'ui/notifications/size-changed', params: { height: Math.ceil(entry.contentRect.height) } }`.
- Host receives: `outerIframe.style.height = `${height}px``.
- No feedback loop possible (host never posts back a height in response).
- The 200ms pre-paint period where the skeleton is shown prevents a visible jump from 320px → real height since the iframe is not visible during loading state.

### MealPlanColumn update after "Pick"

- `commitRecipeToPlan` handler calls `MealPlanStore.addToPlan(...)` (IDB write + pub-sub notify).
- `useMealPlan()` subscriber in `MealPlanColumn` re-renders within one animation frame.
- Newly added plan entry renders as:
  ```
  <li class="py-2 border-b border-gray-100 dark:border-gray-700 last:border-b-0
             text-sm text-gray-700 dark:text-gray-300">
    <span class="font-medium">{recipeId}</span>
    <span class="ml-2 text-gray-400 dark:text-gray-500 text-xs">
      {new Date(addedAt).toLocaleString()}
    </span>
  </li>
  ```
  This matches the existing list item markup already in `MealPlanColumn.tsx`.
- No animation on new entry in Phase 5 (Phase 7 polish if desired).
