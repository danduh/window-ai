<action>
Create `chat/src/app/components/Proofreader/diffSegments.ts` exporting:
1. Type alias `DiffSegment = { type: 'unchanged' | 'removed' | 'inserted'; text: string; correctionTypes?: ProofreaderCorrectionType[]; explanation?: string }`
2. Function `buildDiffSegments(original: string, corrections: ProofreaderCorrection[]): DiffSegment[]`

Algorithm:
1. Defensive copy + sort corrections by `startIndex` ascending
2. Initialize `cursor = 0`, `segments: DiffSegment[] = []`
3. For each correction in sorted order:
   - If `cursor < correction.startIndex`, push `{ type: 'unchanged', text: original.slice(cursor, correction.startIndex) }`
   - Push `{ type: 'removed', text: original.slice(correction.startIndex, correction.endIndex), correctionTypes: correction.types, explanation: correction.explanation }`
   - Push `{ type: 'inserted', text: correction.correction, correctionTypes: correction.types, explanation: correction.explanation }`
   - Update `cursor = correction.endIndex`
4. After loop, if `cursor < original.length`, push final `{ type: 'unchanged', text: original.slice(cursor) }`
5. Return `segments`

Edge cases:
- Empty corrections array → returns single `[{ type: 'unchanged', text: original }]`
- Overlapping corrections → not handled (Proofreader API does not produce overlaps per spec); document with a one-line comment
- Empty `removed` text (pure insertion) → emit `removed` segment with empty text; renderer skips zero-length removed segments

Use TypeScript strict — no `: any`, no `as any`.
</action>
