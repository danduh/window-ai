# Phase 2 — UI Design Contract

> Visual and interaction contract for the in-page WebMCP agent (chat drawer, tool list panel, registration pill, inline tool-call indicators, unavailable state). Generated by gsd-ui-researcher; consumed by gsd-planner, gsd-executor, gsd-ui-checker, gsd-ui-auditor.

**Scope of this UI-SPEC:** new visuals introduced by Phase 2 only. The existing `RecipeWorkbenchPage` chrome (header, gradient icon, Tabs, MissingFlagBanner, RecipePicker, RecipeHeader, IngredientsList, StepsList) was already locked in Phase 1 and is not redefined here. New components attach inside the `Workbench` tab below the recipe view, plus a registration pill in the existing header.

## Design System

| Property | Value |
|----------|-------|
| Tool | none (manual Tailwind tokens) |
| Preset | not applicable — no `components.json` in repo (verified) |
| Component library | none (hand-rolled React + Tailwind) |
| Icon library | inline SVG (existing convention — `viewBox="0 0 24 24"`, `stroke="currentColor"`) |
| Font | Tailwind default sans (system stack) — inherited from existing pages |

**Detected tokens (from `chat/tailwind.config.js`):**
- Custom color scale `primary.50–900` (blue, `#3b82f6` at 500). All accent/CTA usage MUST come from this scale.
- Animations: `animate-slide-up` (300ms ease-out, translateY+opacity), `animate-fade-in` (500ms). Reuse for new chat bubbles and the tool-call indicator.
- Dark mode: `darkMode: 'class'` — every `bg-`, `text-`, `border-` MUST be paired with a `dark:` counterpart (project convention from PROJECT.md and Phase 1 components).

**Reused existing components (no new visual specs needed):**
- `ChatBox` (`chat/src/app/components/ChatBox.tsx`) — fixed `h-96 overflow-y-auto`, `bg-white dark:bg-gray-800 rounded-xl shadow-lg border border-gray-200 dark:border-gray-700`, user bubbles `bg-primary-500 text-white`, assistant bubbles `bg-gray-100 dark:bg-gray-700`. Reused as-is for the chat transcript inside the drawer.
- `ChatInput` (`chat/src/app/components/ChatInput.tsx`) — auto-grow textarea (44–120px), Enter sends / Shift+Enter newline, send button `bg-primary-500 hover:bg-primary-600`. Reused as-is for the drawer's input.
- `MissingFlagBanner` (`chat/src/app/components/RecipeWorkbench/MissingFlagBanner.tsx`) — yellow-50 / yellow-900-20 surface treatment. Mirrored (not reused — different copy + smaller size) for the LanguageModel-unavailable inline state.

## Spacing Scale

Declared values (multiples of 4, mapped to Tailwind utilities):

| Token | Value | Tailwind | Usage in this phase |
|-------|-------|----------|---------------------|
| xs | 4px | `gap-1`, `p-1` | Icon-to-text gaps inside tool-call indicators and pills |
| sm | 8px | `gap-2`, `p-2` | Pill internal padding (`px-2 py-1`), tool-list row spacing |
| md | 16px | `gap-4`, `p-4` | Drawer internal padding, chat bubble padding (existing) |
| lg | 24px | `gap-6`, `p-6` | Card padding for tool list panel + chat drawer card; gap between drawer regions |
| xl | 32px | `gap-8`, `mb-8` | Header bottom margin (existing) |

**Phase 2 exceptions:**
- `min-h-[44px]` on the chat send button and textarea — touch-target floor (existing `ChatInput` convention; do not change).
- Drawer total height: **`h-72` (288px) on `lg+` breakpoints**; **`h-[60vh]` capped at `h-96` on `<lg`** (mobile/tablet) so the recipe view above stays visible (~40% of viewport). Rationale: 288px = ~4 transcript rows + input + tool-call line + small buffer, comfortable on a 1080p laptop while keeping the recipe in view. The viewport-relative cap on small screens preserves the live-update demo when the recipe is short.
- Drawer top margin: `mt-6` (24px) below the WorkbenchPanel grid.

## Color

Project uses a single custom hue (`primary` blue) plus Tailwind's gray/yellow/red/green defaults. The 60/30/10 split below applies to *new* surfaces in this phase.

| Role | Token | Light | Dark | Usage in this phase |
|------|-------|-------|------|--------------------|
| Dominant (60%) | `bg-white` / `bg-gray-800` | `#FFFFFF` | `#1F2937` | Drawer card surface, tool list panel surface, chat bubble (assistant) |
| Secondary (30%) | `bg-gray-50` / `bg-gray-700` | `#F9FAFB` | `#374151` | Tool list row backgrounds, tool-call indicator background, hover/idle states |
| Accent (10%) | `bg-primary-500` / `bg-primary-600` | `#3B82F6` / `#2563EB` | same | Send button, user bubble, registration pill (success state), tool-call active glyph |
| Success | `text-green-700` / `bg-green-50` | `#15803D` / `#F0FDF4` | `text-green-300` / `bg-green-900/20` | `✓ N tools registered` success pill, `✓ done` tool-call completion glyph |
| Warning | `text-yellow-700` / `bg-yellow-50` | `#A16207` / `#FEFCE8` | `text-yellow-300` / `bg-yellow-900/20` | `⚠ 0 tools registered` failure pill, LanguageModel-unavailable inline state (mirrors MissingFlagBanner) |
| Error/Destructive | `text-red-700` / `bg-red-50` | `#B91C1C` / `#FEF2F2` | `text-red-300` / `bg-red-900/20` | Tool-handler-throw inline error in transcript, blocked-response message |

**Accent reserved exclusively for:**
1. Send button (`bg-primary-500 hover:bg-primary-600`) — existing `ChatInput`
2. User chat bubble (`bg-primary-500 text-white`) — existing `ChatBox`
3. The `⚙` glyph color in the in-flight tool-call indicator (`text-primary-600 dark:text-primary-400`)
4. Active tool list item hover state (`hover:bg-primary-50 dark:hover:bg-primary-900/20`) — only when the agent is actively calling that tool (live binding to in-flight tool name)

NOT used for: tool list panel chrome, drawer borders, plain text, registered-state pill (that's green), unavailable banner (that's yellow). This keeps the eye drawn to the live demo moment (assistant typing → tool firing → recipe updating).

**No new destructive actions in this phase.** Phase 2 introduces no delete/destroy/clear flows beyond what exists; tool-handler errors are *informational* (red text) not *destructive* (no confirmation modal). Future destructive actions (e.g., a Clear-chat button) are out of scope.

## Component Inventory (new in Phase 2)

Six new visual components attach to the existing `RecipeWorkbenchPage`. Container hierarchy:

```
RecipeWorkbenchPage (existing)
├── header
│   └── ToolRegistrationPill          ← NEW — renders next to ThemeToggle
├── MissingFlagBanner (existing — !navigator.modelContext)
└── Tabs
    └── Workbench tab
        └── WorkbenchPanel (existing)
            ├── RecipePicker (existing)
            └── recipe view (existing)
        └── AgentDrawer                ← NEW — sticks below WorkbenchPanel
            ├── ToolListPanel          ← NEW — collapsible
            ├── ChatTranscript (= existing ChatBox)
            │   └── ToolCallIndicator  ← NEW — inline system bubble
            ├── ChatInput (existing)
            └── LanguageModelUnavailable ← NEW — inline (mirrors MissingFlagBanner)
```

### 1. ToolRegistrationPill

**Placement:** absolute right side of the existing `<header>`, immediately to the LEFT of `<ThemeToggle />`. Inserted into the existing `flex items-center space-x-3` cluster.

**States:**
| State | Visual | Copy | Tailwind |
|-------|--------|------|----------|
| `idle` (page mounting, before register call) | gray pill, spinner glyph | `Registering tools…` | `bg-gray-100 dark:bg-gray-700 text-gray-700 dark:text-gray-300` |
| `success` (all 8 registered) | green pill, ✓ glyph | `✓ 8 tools registered` | `bg-green-50 dark:bg-green-900/20 text-green-700 dark:text-green-300 border border-green-200 dark:border-green-800` |
| `partial` (n<8 succeeded) | yellow pill, ⚠ glyph | `⚠ {n} of 8 tools registered` | `bg-yellow-50 dark:bg-yellow-900/20 text-yellow-700 dark:text-yellow-300 border border-yellow-200 dark:border-yellow-800` |
| `error` (registration threw) | yellow pill, ⚠ glyph | `⚠ 0 tools registered` | (same as partial) |
| `unavailable` (`!navigator.modelContext`) | hidden | — | `display: none` (the page-level `MissingFlagBanner` carries the message) |

**Geometry:** `inline-flex items-center gap-2 px-3 py-1 rounded-full text-sm font-medium`. Total height 28px (matches existing icon buttons in the header at `p-2 = 32px`).

**Accessibility:** `role="status"` on the wrapper; `aria-live="polite"` so the count announces when registration completes. The glyph is `aria-hidden="true"` (decorative).

**Animation:** fade-in on mount (`animate-fade-in`). No spinner motion in `prefers-reduced-motion`; idle copy alone signals the state.

### 2. AgentDrawer

**Placement:** rendered inside the `Workbench` tab content, below the existing `WorkbenchPanel` grid, separated by `mt-6`. Is NOT a viewport-fixed `position: fixed` overlay — it's an in-flow region anchored to the bottom of the workbench tab. Rationale: avoids covering the recipe view; the page's own scroll keeps both the recipe and the chat live in the same viewport on a 1080p laptop. (CONTEXT.md D-02 says "fixed-height region at the bottom of `RecipeWorkbenchPage`" — this is the chosen interpretation.)

**Card geometry:** `bg-white dark:bg-gray-800 rounded-xl shadow-lg border border-gray-200 dark:border-gray-700 p-6 transition-colors duration-200` (matches every other card on the page). Outer height fixed: `h-72 lg:h-72` desktop, `h-[60vh] max-h-96` on `<lg` (see Spacing exceptions). Internal layout:

```
┌───────────────────────────────────────────────┐
│ ToolListPanel  (collapsed by default)         │  ← shrink-to-fit when open
├───────────────────────────────────────────────┤
│ ChatTranscript (flex-1, overflow-y-auto)      │
│   • assistant: Markdown body                  │
│   • user: primary bubble                      │
│   • ToolCallIndicator: inline system bubble   │
├───────────────────────────────────────────────┤
│ ChatInput (auto-grow 44–120px)                │
└───────────────────────────────────────────────┘
```

Use `flex flex-col gap-3 h-full` on the inner container; ChatTranscript gets `flex-1 min-h-0`. NOTE: the existing `ChatBox` hard-codes `h-96` — for the drawer, render the transcript with a wrapping `<div className="flex-1 min-h-0 overflow-hidden">` and pass `ChatBox` so its scroll behavior still works inside the bounded parent. Acceptable to add a small CSS override (`[&>div:first-child]:h-full`) rather than fork `ChatBox`.

**Heading:** `Recipe agent` — `text-xl font-semibold text-gray-900 dark:text-white mb-3` (only renders if the ToolListPanel is collapsed; when expanded, the panel's own header carries the section).

**Disabled mode (LanguageModel unavailable):** ChatInput is rendered with `disabled={true}` (existing prop greys it out at 50% opacity). The `LanguageModelUnavailable` inline message renders ABOVE ChatInput inside the drawer. ChatTranscript area shows the empty state. Recipe browsing above remains fully interactive.

### 3. ToolListPanel

**Placement:** top region of the AgentDrawer. Mirrors `ToolCallingPage`'s "Available Tools" panel but compact — no checkbox column, no Test button.

**Geometry:**
- Collapsed (default): single row, `flex items-center justify-between`, `bg-gray-50 dark:bg-gray-700 rounded-lg px-4 py-2 cursor-pointer`. Left: heading `Tools (8)` at Label size (14px / weight 500). Right: chevron-down SVG (rotates to chevron-up when expanded).
- Expanded: `bg-gray-50 dark:bg-gray-700 rounded-lg p-4 max-h-40 overflow-y-auto`. Internal: `<ul>` of 8 rows, each `flex items-start gap-2 py-1`. Each row: monospace tool name (`font-mono text-sm font-medium text-gray-900 dark:text-white`) + dim description (`text-sm text-gray-500 dark:text-gray-400`). On `<lg` breakpoints the panel is collapsed by default and the `max-h-40` (160px) cap is reduced to `max-h-32` (128px) so the chat transcript stays usable.

**Live state binding:** the heading reads `Tools (8)` when registration succeeded, `Tools (registering…)` while pending, `Tools (0 — unavailable)` when `!navigator.modelContext`. Live-active tool name (when the agent is mid-call) gets a subtle highlight: that row's background flips to `bg-primary-50 dark:bg-primary-900/20` for the duration of the call.

**Toggle UX:**
- Click anywhere on the collapsed row toggles open/closed.
- Keyboard: `<button type="button" aria-expanded={isOpen} aria-controls="tool-list">` wraps the row. Space/Enter toggles. Tab order: pill → collapse toggle → first tool row (when open) → ChatInput textarea → send button.
- Animation: `transition-[max-height] duration-200 ease-out` when expanding; respects `prefers-reduced-motion: reduce` (drop the height transition, keep the discrete state change).

### 4. ToolCallIndicator

A pseudo-message bubble rendered inside the ChatTranscript, between user and assistant turns, when the agent invokes a tool. It is NOT a new transcript role — it's a "system" bubble injected by the chat handler when a tool-call delta is observed.

**In-flight visual:**
```
┌─────────────────────────────────────────────────────┐
│ ⚙  Calling scaleRecipe(servings: 6)…               │
└─────────────────────────────────────────────────────┘
```
- Container: full-width, `bg-gray-50 dark:bg-gray-700 rounded-lg px-3 py-2 my-2`
- Glyph `⚙`: `text-primary-600 dark:text-primary-400 animate-spin` at 14px (`text-sm`). In `prefers-reduced-motion`, replace `animate-spin` with a static glyph (no rotation).
- Text: `text-sm font-mono text-gray-700 dark:text-gray-300`. Tool name is the full registered name; args are rendered as `(key: value)` for at most 2 keys; if more, render `(…)` and rely on the agent's natural-language summary.

**Completed visual (replaces the in-flight bubble in place):**
```
┌─────────────────────────────────────────────────────┐
│ ✓  scaleRecipe done                                 │
└─────────────────────────────────────────────────────┘
```
- Glyph `✓`: `text-green-600 dark:text-green-400` (no animation).
- Text: `text-sm font-mono text-gray-600 dark:text-gray-400` (dimmer than in-flight to recede visually now that the live moment is over).

**Error visual (handler threw):**
```
┌─────────────────────────────────────────────────────┐
│ ✗  scaleRecipe failed: {message}                    │
└─────────────────────────────────────────────────────┘
```
- Glyph `✗`: `text-red-600 dark:text-red-400`.
- Text: `text-sm font-mono text-red-700 dark:text-red-300`. The agent's subsequent assistant turn explains in natural language; the indicator just marks the failed call.

**Args display rule:** show full arg list if it fits in one line at `text-sm` width inside a 96-char container. Otherwise truncate the value with `…` (CSS `truncate` on a `max-w-[12rem]` span per arg) — never expand the bubble vertically. No full args/result dump (CONTEXT.md D-03).

**Accessibility:** wrap the bubble in `<div role="status" aria-live="polite">` so screen readers announce the call. `aria-atomic="true"` so the in-flight → done transition is read as one update, not two.

### 5. LanguageModelUnavailable (inline)

Mirrors `MissingFlagBanner` styling but inside the drawer and with chat-specific copy. Renders ONLY when `LanguageModel.availability()` returns anything other than `'available'` AND `navigator.modelContext` IS present (otherwise the page-level `MissingFlagBanner` already covers it; don't double-warn).

**Geometry:** `bg-yellow-50 dark:bg-yellow-900/20 border border-yellow-200 dark:border-yellow-800 rounded-lg p-3 text-sm` — same color tokens as `MissingFlagBanner`, smaller padding (`p-3` vs `p-4`), same `role="status"`.

**Content:**
- ⚠ icon (yellow, identical SVG path to `MissingFlagBanner`)
- **Heading line:** `Chrome built-in AI isn't available.` (`font-medium text-yellow-800 dark:text-yellow-200`)
- **Body line:** `The recipe tools are still registered (try the Tool Inspector). To run the in-page agent, enable Chrome built-in AI in Chrome 146+ Canary.` (`text-yellow-700 dark:text-yellow-300 mt-1`)
- No flag table (the page-level `MissingFlagBanner` already shows it; the drawer message is just a contextual explainer, not a duplicate instructions card).

When this state shows, ChatInput is `disabled` and ChatTranscript shows the empty state copy below.

### 6. ToolListPanel — empty/error sub-states

Inherits the ToolListPanel container but content varies:
- `unavailable`: shows a single dim row `WebMCP not enabled in this browser.` (`text-sm text-gray-500 dark:text-gray-400 italic`). The 8-tool list is hidden — there's nothing to show.
- `partial` (n<8): renders all 8 tool names; failed tools get an `aria-label="Failed to register"` annotation and a `text-red-600` exclamation glyph next to the name.

## Accessibility Contract

| Requirement | How |
|-------------|-----|
| Live transcript announces new assistant tokens | The ChatTranscript wrapper carries `role="log" aria-live="polite" aria-relevant="additions"`. Don't put `aria-live` on each bubble — that produces double-announcements with streaming. |
| Tool-call indicator announces when fired | Indicator wrapper: `role="status" aria-live="polite" aria-atomic="true"`. The full `Calling {tool}…` string is announced once; the transition to `done` is announced as a state update on the same node. |
| Registration pill announces state changes | Pill wrapper: `role="status" aria-live="polite"`. Idle → success transition is announced once. |
| Keyboard tab order | (1) registration pill — focusable only if interactive; pill is informational so `tabindex` is omitted. (2) ToolListPanel collapse toggle — `<button type="button" aria-expanded aria-controls>`. (3) ChatInput textarea — existing focusable. (4) Send button — existing focusable. (5) Skip-back-to-top: not introduced this phase. |
| Reduced motion | `animate-spin` on the in-flight gear glyph degrades to a static `⚙` (CSS `@media (prefers-reduced-motion: reduce) { .animate-spin { animation: none; } }`). `animate-pulse` on the awaiting-first-token dots degrades to a static `…`. Drawer expand/collapse `transition-[max-height]` degrades to instant. The streaming text caret (if any) is NOT animated — incremental text alone signals progress. |
| Disabled chat input has clear affordance | `disabled` prop already applies `opacity-50 cursor-not-allowed` (existing). Plus: when LanguageModel is unavailable, the LanguageModelUnavailable card sits directly above the input — visual context for why it's disabled. |
| Color is not the only signal | Pill states pair color with glyph (`✓` success, `⚠` warning). Tool-call states pair color with glyph (`⚙` / `✓` / `✗`). Unavailable inline pairs color with `⚠` icon. Tool-handler errors include `failed:` text alongside red color. |
| Focus visible | All new buttons inherit `focus:outline-none focus:ring-2 focus:ring-primary-500` from existing patterns; do not remove the focus ring. |
| Tool list scrollable region | When expanded with `max-h-40 overflow-y-auto`, the `<ul>` carries `tabindex="0"` so keyboard users can scroll with arrows when focus lands inside. |

| Breakpoint | AgentDrawer | ToolListPanel | ChatInput | Notes |

If the demo viewport feels cramped during execution, the executor MAY revisit by switching to a `lg:sticky lg:bottom-0` pattern on the drawer — record as a deviation if so.

## Pill Placement Decision

Per CONTEXT.md "Claude's Discretion": pill goes **in the existing header, immediately before `<ThemeToggle />`**, not in the Tabs row.

Rationale:
1. The page header is the highest-attention region — visible on every page state including unavailable browsers (the page-level banner is below the header, so the pill can't be hidden by it).
2. The Tabs row is already crowded with two tabs (Workbench / Docs) on a max-w-6xl container; a status pill there would compete for attention with active-tab styling.
3. ThemeToggle is small (32px) — pill at 28px sits visually balanced beside it without breaking the existing `space-x-3` rhythm.

Implementation: insert one new flex child before `<ThemeToggle />` in `RecipeWorkbenchPage.tsx`'s header `<div className="flex items-center space-x-3">`. No structural change to the header.
