## Spacing Scale

Declared values (multiples of 4 — matches Tailwind default scale as used throughout the codebase):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps (`space-x-1`), badge padding |
| sm | 8px | Input internal padding-y (`p-2`), button padding-y |
| md | 16px | Default element spacing (`p-4`), card internal padding (`p-4`) |
| lg | 24px | Section spacing (`p-6`), header margin-bottom (`mb-6`) |
| xl | 32px | Header margin-bottom on ProofreaderHeader (`mb-8`) |
| 2xl | 48px | Not used in demo pages — available for layout breaks |
| 3xl | 64px | Not used in demo pages |

Exceptions:
- Page outer wrapper: `max-w-6xl mx-auto p-4` (16px all sides) — locked pattern from CONTEXT.md and all existing demo pages.
- Touch targets / click affordances: minimum 44x44px per accessibility convention (applied to send button, remove-image button).
- Header icon badge: `w-12 h-12` (48px) — mirrors `ProofreaderHeader` and `GenerativeUIHeader`.

## Typography

All values are extracted directly from the existing component codebase. Phase 10 must not introduce new sizes or weights.

| Role | Size | Weight | Line Height | Tailwind class |
|------|------|--------|-------------|----------------|
| Page heading (h1) | 30px | 700 (bold) | 1.2 (tight) | `text-3xl font-bold leading-tight` |
| Subheading / label | 16px | 500 (medium) | 1.5 | `text-base font-medium` |
| Body / description | 16px | 400 (normal) | 1.625 (relaxed) | `text-base` |
| Small / meta | 14px | 400 (normal) | 1.5 | `text-sm` |
| Micro / caption | 12px | 400 (normal) | 1.4 | `text-xs` |
| Nav links | 14px | 500 (medium) | — | `text-sm font-medium` (desktop nav) |
| Mobile nav links | 16px | 500 (medium) | — | `text-base font-medium` (mobile nav) |

Declared as 5 sizes, 2 weights (400 normal + 500/700 semibold/bold). Weights are `font-medium` for interactive labels and `font-bold` for headings; body is weight 400.

Source: `ProofreaderHeader.tsx` (h1 text-3xl font-bold), `ProofreaderForm.tsx` (label text-sm font-medium), `AppRouter.tsx` nav links (text-sm font-medium / text-base font-medium), `ChatBox.tsx` prose body, `ProofreaderPage.tsx` footer (text-xs).

## Color

Primary color family: `primary-*` = blue scale (`#3b82f6` at 500, `#2563eb` at 600, `#1d4ed8` at 700).

| Role | Tailwind tokens | Hex reference | Usage |
|------|-----------------|---------------|-------|
| Dominant (60%) | `bg-white` / `dark:bg-gray-800` | #ffffff / #1f2937 | Page background, panel backgrounds |
| Secondary (30%) | `bg-gray-100` / `dark:bg-gray-700` | #f3f4f6 / #374151 | Assistant bubbles, input fields, card surfaces |
| Accent (10%) | `bg-primary-500` / `bg-primary-600` | #3b82f6 / #2563eb | User chat bubbles, send button, focus rings, progress bar fill, drag-over border |
| Destructive | `text-red-600` / `dark:text-red-400` | #dc2626 / #f87171 | Inline error messages (image MIME rejection, promptWithImage errors) |
| Warning / banner | `bg-yellow-50` / `dark:bg-yellow-900/20`, `border-yellow-200` | — | MissingFlagBanner background — reuses existing component styling exactly |

Accent reserved for EXACTLY these elements:
1. User chat bubbles (`bg-primary-500 text-white`)
2. Send button background (`bg-primary-600 hover:bg-primary-700`)
3. Focus rings on all interactive inputs (`focus:ring-primary-500`)
4. Download progress bar fill (`bg-primary-500`)
5. Drag-over panel border (`border-primary-500`, dashed variant)
6. Active tab underline (matches existing `Tabs.tsx` implementation)

All other interactive elements (disabled states, secondary buttons) use gray scale, not accent.

Dark mode: all components use `dark:` variants consistently with the existing pattern — `dark:bg-gray-800`, `dark:text-gray-100`, `dark:border-gray-600`, `dark:placeholder-gray-400`.

Source: `chat/tailwind.config.js` color scale, `ChatBox.tsx` bubble classes, `ProofreaderForm.tsx` button classes, `MissingFlagBanner.tsx` banner classes, `global.css` `.focus-ring` utility.

## Component Specifications

This section prescribes the visual contract for each net-new component in Phase 10.

### MultimodalPage (page shell)

```
min-h-screen bg-white dark:bg-gray-800 transition-colors duration-200
  └─ max-w-6xl mx-auto p-4
       ├─ MissingFlagBanner (conditional, renders above header when pageState === 'unavailable')
       ├─ MultimodalHeader
       ├─ Tabs (Docs first, Chat second)
       └─ footer: mt-4 text-center text-xs text-gray-500 dark:text-gray-400
                  "🔒 Zero network during demo — open DevTools → Network tab"
```

Pattern source: `ProofreaderPage.tsx` line 220–249 (identical shell structure).

### MultimodalHeader

```
<header className="mb-8">
  <div className="flex items-center space-x-4">
    Icon badge: w-12 h-12 rounded-xl bg-gradient-to-br from-primary-500 to-purple-600
                flex items-center justify-center text-white shadow-sm
    Icon: camera SVG w-6 h-6 (inline, aria-hidden)
    h1: text-3xl font-bold text-gray-900 dark:text-white leading-tight
        Content: "Multimodal"
    p: mt-2 text-base text-gray-600 dark:text-gray-400
       Content: "On-device image understanding with Gemini Nano — drag, paste, or capture an image and ask about it. Chrome 148+ stable, zero network."
```

Pattern source: `ProofreaderHeader.tsx` (pixel-identical structure, different icon + copy).

### MultimodalChatPanel (chat panel container)

```
bg-white dark:bg-gray-800 rounded-xl shadow-lg border border-gray-200 dark:border-gray-700
p-6 transition-colors duration-200 h-full min-h-[60vh] lg:min-h-[600px] flex flex-col
  └─ flex flex-col gap-3 h-full min-h-0
       ├─ MultimodalTranscript (flex-1 min-h-0 overflow-y-auto)
       ├─ download progress bar (conditional, pageState === 'downloading')
       └─ MultimodalInput
```

Drag-over state (applied to the panel container):
- Border changes from `border-gray-200 dark:border-gray-700` to `border-2 border-dashed border-primary-500`
- Translucent overlay inside: `absolute inset-0 bg-primary-50/80 dark:bg-primary-900/40 rounded-xl flex items-center justify-center z-10`
- Overlay label: `text-primary-700 dark:text-primary-300 text-lg font-medium` — "Drop image here"
- `dragenter`/`dragleave` ref counter prevents flicker (see CONTEXT.md specifics section)

Pattern source: `GenUIChatPanel.tsx` outer container classes + CONTEXT.md drag-over decision.

### MultimodalTranscript

Scroll behavior: `overflow-y-auto`, auto-scrolls to bottom on new message (`messagesEndRef.current?.scrollIntoView({ behavior: 'smooth' })`).

Empty state:
```
flex items-center justify-center h-full text-gray-500 dark:text-gray-400
  └─ text-center
       ├─ camera/image SVG: w-12 h-12 mx-auto mb-4 opacity-50
       └─ p: "Drop an image or paste (⌘V) to get started"
```

Message bubbles (user):
```
flex justify-end animate-slide-up
  └─ max-w-[80%] p-4 rounded-2xl shadow-sm bg-primary-500 text-white break-words
       ├─ (when attachedImageUrl present) img: max-w-full rounded-lg mb-2 max-h-48 object-cover
       │   — image renders ABOVE the text in the same bubble
       └─ div.prose.prose-sm.prose-invert.max-w-none (Markdown text)
```

Message bubbles (assistant):
```
flex justify-start animate-slide-up
  └─ max-w-[80%] p-4 rounded-2xl shadow-sm bg-gray-100 dark:bg-gray-700
       text-gray-800 dark:text-gray-200 break-words
       ├─ (when streaming) "Thinking…" placeholder with animated ellipsis:
       │   span.animate-pulse text-gray-500 dark:text-gray-400 — "Thinking…"
       └─ div.prose.prose-sm.dark:prose-invert.max-w-none (Markdown text, streams in)
```

Error bubble (assistant, on promptWithImage throw):
```
Same assistant bubble shape, content:
  ├─ p: text-red-600 dark:text-red-400 text-sm — "Couldn't process image — {error message}"
  └─ button: mt-2 text-xs text-primary-600 dark:text-primary-400 underline — "Retry"
```

Pattern source: `ChatBox.tsx` bubble classes (pixel-identical for standard bubbles).

### MultimodalInput (input area)

```
border border-gray-200 dark:border-gray-700 rounded-xl p-3
  ├─ (when pendingImage) thumbnail preview row:
  │     relative inline-block mb-2
  │       img: w-20 h-20 object-cover rounded
  │       button ×: absolute -top-2 -right-2 w-5 h-5 rounded-full
  │                 bg-gray-800 dark:bg-gray-200 text-white dark:text-gray-800
  │                 text-xs flex items-center justify-center
  │                 hover:bg-red-600 dark:hover:bg-red-400 transition-colors
  │       (optional) p: text-xs text-gray-500 dark:text-gray-400 mt-1 truncate max-w-[80px]
  │                    — filename label, show if trivial to include
  ├─ textarea:
  │     w-full resize-none bg-transparent outline-none
  │     text-gray-900 dark:text-gray-100 placeholder-gray-500 dark:placeholder-gray-400
  │     text-sm min-h-[48px] max-h-[120px] overflow-y-auto
  │     Placeholder: "Drop an image or paste (⌘V) — then ask me about it"
  └─ send button row: flex justify-end mt-2
         button: flex items-center gap-2 bg-primary-600 hover:bg-primary-700
                 disabled:bg-gray-400 dark:disabled:bg-gray-600
                 text-white px-4 py-2 rounded-lg font-medium text-sm
                 disabled:cursor-not-allowed transition-colors duration-200
                 — Content: paper-plane SVG (w-4 h-4) + "Send" text (mirrors v1.0/v1.1 pattern)
```

Send button disabled states:
- `pageState !== 'ready'` — tooltip: "Enable multimodal image input to use this demo"
- `pendingImage === null` — tooltip: "Attach an image first"
- `text.trim().length === 0` — tooltip: "Type a question about the image"
- `pageState === 'prompting'` — button shows "Sending…" text, spinner SVG replaces paper-plane

Tooltip implementation: HTML `title` attribute (no custom tooltip library — keeps it simple for demo).

### Inline error (MIME rejection)

```
p: mt-2 text-sm text-red-600 dark:text-red-400
   "Only JPEG, PNG, or WebP images supported"
```
Auto-clears after 4s or on next successful attach. Rendered below the input area inside `MultimodalInput`.

### Download progress bar

```
div.mt-4
  ├─ div: h-2 bg-gray-200 dark:bg-gray-700 rounded overflow-hidden
  │    └─ div: h-full bg-primary-500 transition-[width] duration-300
  │            style={{ width: `${downloadPct}%` }}
  └─ p: mt-1 text-xs text-gray-500 dark:text-gray-400
        "Downloading multimodal model — {pct}%"
```

Pattern source: `ProofreaderPage.tsx` lines 171–183 (pixel-identical, different label text).

## Tabs Configuration

```tsx
<Tabs basePath="/multimodal" defaultTab="docs" tabs={[
  { id: 'docs',  label: 'Docs', path: '/docs', content: docsContent },
  { id: 'chat',  label: 'Chat', path: '',      content: chatContent  },
]} />
```

Docs tab FIRST — v1.1 + Phase 8 lesson: `Tabs.tsx` matches via `currentPath.includes(tab.path)`; the empty-path chat tab matches everything, so docs must come first.

## Interaction States Summary

| State | Visual Signal |
|-------|--------------|
| pageState idle | Chat panel visible, send button disabled (no image attached yet) |
| pageState unavailable | MissingFlagBanner above header; chat input disabled with tooltip |
| pageState downloading | Progress bar + "Downloading…" label; send button shows "Download model first" |
| pageState ready (no image) | Input active, send disabled with "Attach an image first" tooltip |
| pageState ready (image, no text) | Thumbnail preview shown, send disabled with "Type a question" tooltip |
| pageState ready (image + text) | Send button enabled, primary-600 background |
| pageState prompting | Send button shows "Sending…" + spinner; input disabled |
| pageState error | Error bubble with "Retry" link in transcript |
| drag-enter panel | Dashed primary-500 border, translucent overlay with "Drop image here" |
| drag-leave panel | Border and overlay revert |
| image attached | Thumbnail w-20 h-20 object-cover rounded with × removal button |
| streaming response | Assistant bubble text accumulates chunk by chunk |
