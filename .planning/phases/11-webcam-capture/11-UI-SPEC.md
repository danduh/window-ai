## Accessibility Notes

All Phase 11 interactive elements follow Phase 10's established accessibility pattern:

- All buttons have programmatic labels (via visible text or `aria-label`)
- Dismiss √ó on error cards: `aria-label="Dismiss camera error"` (required ‚Äî icon-only button)
- `<video>` element: `aria-hidden` is NOT appropriate (it conveys functional information); add `aria-label="Webcam live preview"` on the video container div
- Perf badge: `aria-live="off"` (rapid updates ‚Äî screen reader should not announce every frame)
- All focus rings use `focus:ring-primary-500 focus:ring-offset-2` (inherited from Phase 10)
- Error cards: role card is implicit; heading + body provide sufficient context
- `<video>`: `playsInline muted autoPlay` ‚Äî muted is required for autoplay without user gesture
